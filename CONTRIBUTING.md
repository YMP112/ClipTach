# Contributing to ClipTach

## עקרונות

- לשמור על פשטות, קריאות ותחזוקה קלה.
- להימנע מפיצ'רים שלא קשורים ישירות לחילוץ אובייקט ומסכה.
- להעדיף שינויים קטנים וברורים על פני refactor רחב ולא ממוקד.

## תהליך עבודה

1. לפתוח Issue (או לבחור Issue קיים).
2. ליצור branch ייעודי.
3. לבצע שינוי ממוקד.
4. להריץ:

```bash
flutter format .
flutter analyze
flutter test
```

5. לפתוח Pull Request עם תיאור ברור.

## הנחיות קוד

- קבצים קטנים יחסית, אחריות ברורה לכל מודול.
- להימנע מ"קבצי אלוהים" (God files).
- להוסיף הערות רק כשיש לוגיקה לא טריוויאלית.
- לשמור על API פנימי עקבי ופשוט.
